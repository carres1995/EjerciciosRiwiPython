Qué es el paradigma funcional?

Es un estilo de programación donde:

✔️ El código se construye principalmente con funciones

Las funciones reciben datos y devuelven resultados sin modificar nada fuera de ellas.

✔️ Las variables no cambian (inmutables)

Una vez creado un valor, no se altera. En lugar de cambiarlo, se crea uno nuevo.

✔️ No hay efectos secundarios

Una función funcional no debería imprimir, no debería modificar variables globales, ni escribir en archivos.
Solo recibe datos y devuelve datos.

✔️ Las funciones son tratadas como valores

Puedes guardarlas en variables, pasarlas como argumentos y devolverlas desde otras funciones.

⸻

✅ ¿Para qué sirve?

El paradigma funcional ayuda a:

▶️ 1. Hacer el código más fácil de entender

Si una función siempre hace lo mismo con los mismos datos, es más predecible.

▶️ 2. Evitar errores

Como los datos no cambian, es más difícil que se dañen por accidente.

▶️ 3. Hacer programas más fáciles de probar

Una función pura es muy sencilla de testear.

▶️ 4. Facilitar la programación paralela

Como no se modifican datos compartidos, es más seguro ejecutar tareas en paralelo o concurrentemente.

▶️ 5. Escribir código más corto
